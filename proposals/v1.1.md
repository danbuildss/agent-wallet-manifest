# Proposal: Agent Wallet Manifest v1.1

**Status:** Draft  
**Opened:** 2026-06-27  
**Author:** Zetta AI  

---

## Summary

v1.1 is a non-breaking minor version. All existing v1 manifests remain valid. All new fields are optional.

This proposal covers:
1. Field renames with legacy alias deprecation
2. Optional wallet-level `proof` stub for future attestations
3. Optional root-level `tags`
4. `$id` alignment

---

## 1. Field renames

### `x` → `x_handle`

**Current state:** Schema has `x`. Docs always used `x_handle`. Both are now in the schema as aliases.

**v1.1 plan:** Formally deprecate `x`. Keep it valid. Add deprecation notice in schema description. Remove in v2.

### `fee_recipient` → `fee_receiver`

**Current state:** Schema had `fee_recipient`. Docs always used `fee_receiver`. Both are now in the role enum as aliases.

**v1.1 plan:** Formally deprecate `fee_recipient`. Keep it valid. Add deprecation notice in schema description. Remove in v2.

---

## 2. Wallet-level `proof` stub

Future attestations, signatures, and ZK proofs need a clean place in the schema. Adding an optional, open-ended `proof` object now prevents a breaking change later.

**Proposed addition to each wallet object:**

```json
"proof": {
  "type": "object",
  "description": "Optional cryptographic proof of wallet ownership. Reserved for future attestation standards (signatures, DIDs, ZK proofs). Structure is intentionally open-ended.",
  "properties": {
    "type": {
      "type": "string",
      "description": "Proof type identifier (e.g. 'eip191', 'did_assertion', 'zk_proof')."
    },
    "value": {
      "type": "string",
      "description": "The proof value or reference."
    },
    "created": {
      "type": "string",
      "format": "date",
      "description": "Date the proof was created."
    }
  },
  "additionalProperties": true
}
```

This is intentionally loose — `additionalProperties: true` — so proof systems can evolve without schema changes.

---

## 3. Optional `tags` at root level

Allows ecosystems and frameworks to tag manifests for discovery without requiring a schema change per use case.

**Proposed addition to root object:**

```json
"tags": {
  "type": "array",
  "description": "Optional tags for ecosystem categorization and discovery.",
  "items": {
    "type": "string",
    "maxLength": 50
  },
  "maxItems": 20
}
```

Examples: `["agentkit", "defi", "base-native"]`

---

## 4. `$id` alignment

v1.1 schema `$id` will be `https://schema.zettaai.co/wallets.v1.1.json`. The v1 `$id` is already set to `https://schema.zettaai.co/wallets.v1.json` as of the neutrality update.

---

## Breaking changes

None. All changes are additive or deprecation notices on fields that remain valid.
